Exercise 0 - Setup

Make sure you have a working development environment for the exercises. You will need the following:

  • Apache web server running with support for PHP (e.g. see MAMP)
  • Text editor with syntax highlighting for html, javascript and php (e.g. Sublime Text, Textmate or TextWrangler)
  • An up-to-date browser (Firefox, Chrome, Safari)
Now test your setup:
  1. Locate your webserver's document root, i.e. the place on the filesystem from which the html and php documents will be served (e.g. /Applications/MAMP/htdocs)
  2. Download and unpack the exercises into the webserver's document root (zip or tar.gz)
  3. Open up test.php in your browser, i.e. by navigating to http://localhost/sae.ajax/test.php.
  4. Note 1: the actual URL may differ depending on your installation
  5. Note 2: you should not open the file directly from the filesystem, i.e. file:///Applications/MAMP/htdocs/sae.ajax/test.php
  6. You should see a page containing only one line, e.g.: hello, world! running PHP 5.5.14

Exercise 1 - Using the Developer Console to Debug Javascript

Something's wrong with this html file, have a look at it: level01/exercise01.html

Find the bug in the javascript code and fix it. Proceed as follows:

  1. Open the file from your webserver
  2. Test the application, something's not working...
  3. Open your browser's development console (see here for Firefox, Chrome or Safari)
  4. Alternative: install the Firebug Plugin and open it
  5. Interpret the errors messages in the console
  6. Figure out what's wrong and fix the code!

Exercise 2 - Code a Small Javascript Game

Write a JavaScript program where the program takes a random integer between 1 to 10, the user is then prompted to input a guess number. If the user input matches with guess number, the program will display a message "Good Work" otherwise display a message "Not matched".

  • Don't start from scratch, use html_template.html as a start
  • You can use the expression Math.ceil(Math.random() * 10) to generate a random number in the range 1-10
  • Use prompt() to get input from the user and alert() to output messages.
  • Pay good attention to messages in the developer console while testing your code

Exercise 3 - Analyzing Simple AJAX Requests with the Developer Console

Have a look at the following files from your webserver:

  • level01/exercise02.php
  • level01/exercise02.html
Open the html file, play with the application, make sure that it works correctly (the app tells some superheroes' secret identities).

  1. Open your browser's console, switch to the Network tab and analyse what happens when you click on the button.
  2. What kind of requests are sent by the browser?
  3. Which parameter is used to query a superhero's identity?
  4. How does the server format the response? (hint: look at the Content-Type header)

    5. Exercise 4 - Analyzing Real World AJAX

    1. Open your browser and navigate to Google
    2. Open the development console under the Network tab
    3. Type in some characters into the Google search bar and watch how suggestions appear
    4. Analyse the AJAX requests in the console
    5. What kind of requests are sent by the browser?
    6. In which parameter are the characters transmitted from the browser to Google?
    7. How are the suggestions transmitted back from Google to the browser?
